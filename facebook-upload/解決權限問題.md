# Facebook 權限問題解決方案

## 問題診斷

目前的 Token 缺少 `pages_manage_posts` 權限，導致無法上傳照片。

## 解決方案

### 方案 1：使用你自己的 App（推薦）

1. 在 Graph API Explorer 頁面頂部
2. 點擊 **應用程式下拉選單**（目前可能顯示 "Graph API Explorer"）
3. 選擇你建立的 App（例如 "Photo Uploader" 或你之前建立的 App）
4. 點擊「**產生存取權杖**」
5. **務必勾選**：
   - ✓ `pages_manage_posts` ← 最重要！
   - ✓ `pages_read_engagement`
   - ✓ `public_profile`
6. 點擊「繼續」並授權
7. 在請求欄位輸入 `/me/accounts`
8. 點擊「送出」
9. 複製回應中「賜安診所」的 `access_token`

### 方案 2：檢查 App 模式

如果你的 App 還在「開發模式」：

1. 前往 https://developers.facebook.com/apps/
2. 選擇你的 App
3. 進入「設定」→「基本設定」
4. 檢查「應用程式模式」
5. 如果是「開發中」，可能需要切換到「正式」（但需要審核）

### 方案 3：使用 curl 或 PowerShell 直接測試

如果 Graph API Explorer 有問題，可以直接用命令測試：

```powershell
# 測試上傳
$token = "你的 Token"
$pageId = "865296870210875"
$imagePath = "H:\Gemini\pss\facebook-upload\temp\班表.jpg"
$message = "📅 115 年 6 月醫師門診輪班表【班表會視需要隨時更動與更新】"

$form = @{
    access_token = $token
    message = $message
    source = Get-Item $imagePath
}

Invoke-RestMethod -Uri "https://graph.facebook.com/v18.0/$pageId/photos" -Method Post -Form $form
```
