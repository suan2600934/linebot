# 賜安診所 LINE 驗證碼系統

## 功能
櫃檯查詢病人資料、產生 6 位 LINE 綁定驗證碼、查詢/解除既有綁定。

---

## 檔案結構
```
lineid_code/
├── patdb_query.py      # Tkinter GUI 主程式（櫃台端使用）
```

## 首次安裝（在掛號電腦上安裝 Python）

### 步驟 1：安裝 Python 3
1. 從 https://www.python.org/downloads/ 下載 Python 3.8 或更新版本
2. **安裝時務必勾選「Add Python to PATH」**（重要！）
3. 安裝完成後開啟命令列（按 `Win+R`，輸入 `cmd`，按 Enter）

### 步驟 2：安裝所需套件
在命令列輸入：
```
pip install dbfread requests
```

### 步驟 3：確認網路硬碟路徑
向資訊人員確認 patdb.dbf 的完整 UNC 路徑，例如：
- `\\192.168.1.100\clinic\patdb.dbf`
- `\\看診電腦名稱\C$\CLINIC\DATA\patdb.dbf`

### 步驟 4：修改設定檔
用記事本開啟 `config.json`，修改 `patdbPath`：
```json
{
  "patdbPath": "\\\\192.168.1.100\\clinic\\patdb.dbf",
  "apiBaseUrl": "https://lineid-code.zeabur.app",
  "APP_KEY_V1": "base64-32-bytes-key==",
  "UNBIND_API_KEY": "櫃台強制解除金鑰",
  "logDir": "logs"
}
```
**注意**：`\\` 是 JSON 格式要求，路徑中每個 `\` 要打成 `\\`。

### 步驟 5：測試執行
在命令列進入本資料夾，執行：
```
python patdb_query.py
```
看到「已載入 X 筆資料」表示成功。

### 步驟 6：建立捷徑（可選）
在桌面上建立 `patdb_query.py` 的捷徑，未來直接點兩下就能開。

---

## 換電腦後的設定

新掛號電腦需要做的動作：
1. 重複「步驟 1、2」（已安裝過可跳過）
2. 把本資料夾複製到新電腦（含 `bindings.db` 與 `logs/`）
3. 修改新電腦上的 `config.json`，確認 `patdbPath`、`APP_KEY_V1`、`UNBIND_API_KEY` 正確
4. 執行 `python patdb_query.py` 確認正常

## 操作流程（驗證碼產生）

### 初始畫面
開啟 `patdb_query.py` 後共有兩個分頁：
- **驗證碼產生**：產生 LINE 綁定驗證碼
- **綁定管理**：查詢本地綁定記錄、強制解除

### 完整流程（從頭到尾）

#### 1. 輸入綁定人（A）
- 在「【A】綁定人」區塊的搜尋框輸入**姓名 / 身分證 / 生日**任一關鍵字
- 按 **Enter** 或點「查詢」
- 結果列表會顯示符合的病人
- **雙擊**或點「選擇此人為綁定人」
- 下方會顯示「已選擇：xxx（ID / 生日）」

#### 2. 輸入被綁定人（B）
- 在「【B】被綁定人（要看診的病人）」區塊用相同方式搜尋、雙擊選擇
- 確認顯示「已選擇：xxx」

#### 3. 點「確認並產生驗證碼」
- 當 A 和 B 都選擇後，「確認並產生驗證碼」按鈕由灰色變成可點擊
- 按鈕下方的綠色文字顯示「【A】xxx 將綁定 【B】yyy（RECNO）」
- 點按鈕後：
  1. **API 呼叫**：`POST /api/create-verify-code` 產生 6 位驗證碼（5 分鐘有效）
  2. **重複綁定檢查**：若 A 曾綁過該 RECNO，會跳出警告（如已綁過某人，需先取消）
  3. **檢查 LINE 綁定狀態**：
     - 若 LINE 已有該 RECNO 綁定 → **直接寫入本地資料庫**，跳出「LINE 已有有效綁定，本地記錄已儲存！」
     - 若 LINE 還沒綁定 → 啟用「確認 LINE 綁定」按鈕（**藍色**），顯示紅色文字「等待 LINE 綁定確認：A → B」
  4. **驗證碼區塊**顯示 6 位數字（紅色大字），下方是「有效期至：XXX」（5 分鐘）
  5. 驗證碼會**自動複製到剪貼簿**
  6. 跳出訊息框：「請告訴『A』到 LINE 輸入驗證碼，完成後回來點『確認 LINE 綁定』」

#### 4. 病人去 LINE 輸入驗證碼
- 病人打開 LINE，找到「賜安診所」官方帳號
- 在對話中輸入 6 位驗證碼
- LINE Bot 呼叫 `POST /api/verify` 驗證 → 成功後在 Supabase `line_user_links` 建立連結
- 病人會收到「綁定成功」訊息

#### 5. 櫃台點「確認 LINE 綁定」
- 回到 `patdb_query.py`，點已變為藍色的「確認 LINE 綁定」按鈕
- 系統呼叫 `GET /api/admin/links-by-recno-hash` 查詢 LINE 是否已建立綁定
  - **若已綁定** → 寫入本地 SQLite `binding_records`，跳出「LINE 綁定成功，本地記錄已儲存！」，按鈕恢復灰色
  - **若還沒綁定** → 跳出「LINE 尚未完成綁定，請確認已在 LINE 輸入驗證碼後再點」

#### 6. 已完成綁定
- 在「綁定管理」分頁可以看到所有 active 綁定記錄

## 操作流程（綁定管理 - 解除綁定）

### 查詢綁定記錄
- 切換到「綁定管理」分頁
- 選擇查詢方式（全部 / 依綁定人 / 依被綁定人 / 依時間）
- 輸入關鍵字（姓名 / 身分證 / 生日 / RECNO / 時間任一字元符合即可）
- 列表顯示 `【A】xxx 綁定 【B】yyy | RECNO：xxx | 綁定時間：xxx`

### 強制解除綁定
- 在列表中**選擇**一筆記錄
- 點「解除綁定」按鈕
- 跳出確認框顯示 A、B、RECNO、時間
- 按「是」確認
- 系統流程：
  1. 呼叫 `GET /api/admin/links-by-recno-hash` 查 LINE 連結
  2. 若找到 → 呼叫 `POST /api/admin/unbind` 解除綁定 → 更新本地狀態為 `unbound`
  3. 若 LINE 找不到（資料不一致）→ 提示「雲端無有效綁定記錄，本地記錄已移除」並更新本地狀態

### 重複綁定處理
若嘗試對同一 binder（A）已綁過的 RECNO 再產生驗證碼：
- 跳出警告「重複綁定：A 已經綁定過 B（RECNO：xxx）」
- 引導到「綁定管理」先解除舊綁定

## UI 狀態與按鈕變化

### 「確認並產生驗證碼」按鈕（Tab 1 中段）

| 狀態 | 按鈕顏色 | 文字 | 觸發動作 |
|------|---------|------|---------|
| A 或 B 尚未選 | 灰色（disabled） | 「確認並產生驗證碼」 | - |
| A 和 B 都選好 | 一般（可點） | 「確認並產生驗證碼」 | 觸發 `generate_code()` |
| 驗證碼產生後 | 一般（可點） | 「確認並產生驗證碼」 | 可再次點擊重新產生 |

### 「確認 LINE 綁定」按鈕（驗證碼區塊內）

| 狀態 | 按鈕樣式 | 文字 | 狀態標籤 |
|------|---------|------|---------|
| 初始 / 已清除 | 灰底灰字（`ConfirmLineDisabled.TButton`） | 「確認 LINE 綁定」 | 空白 |
| 等待 LINE 完成 | 藍底白字粗體（`ConfirmLine.TButton`） | 「確認 LINE 綁定」 | 紅色粗體「等待 LINE 綁定確認：A → B」 |
| LINE 已綁定成功 | 灰底灰字 | 「確認 LINE 綁定」 | 空白 |

### 「驗證碼」區塊（LabelFrame）

| 狀態 | 6 位數字 | 有效期限 | 複製按鈕 |
|------|---------|---------|---------|
| 未產生 | 「（尚未產生）」（紅字） | 空白 | 灰色 |
| 已產生 | 6 位數字（紅字大字） | 「有效期至：2026-07-11 21:30:00（5 分鐘）」 | 可點 |

### 「驗證碼」LabelFrame 內的版面順序（**重要：視窗太小需滾動**）

```
驗證碼 ─────────────────────
   123456         ← 6 位數字（紅字大字）
   有效期至：xxx
   [複製驗證碼]
   [確認 LINE 綁定]    ← 啟用後變藍底白字
   等待 LINE 綁定確認：A → B   ← 紅字（啟用時）
```

視窗在 1080p 以下解析度時，「驗證碼」區塊可能需**向下滾動**才能看到。

## 常見問題

**Q: 出現「patdb 檔案不存在」**
→ 檢查 `config.json` 中的路徑是否正確，確認看診電腦已開機且網路可連線。

**Q: 出現「無法連線到 API 伺服器」**
→ 確認網路正常。需允許對 `lineid-code.zeabur.app:443` 的 HTTPS 連線。

**Q: 出現「重複綁定」警告**
→ A 已綁過該 RECNO。請切到「綁定管理」先解除舊綁定。

**Q: 出現「LINE 已有有效綁定，本地記錄已儲存！」**
→ 表示輸入驗證碼時已經完成 LINE 綁定，系統自動偵測並寫入本地資料庫，無需再點「確認 LINE 綁定」。

**Q: 按下「確認 LINE 綁定」出現「LINE 尚未完成綁定」**
→ 表示 LINE 還沒完成綁定。請確認病人在 LINE 已輸入驗證碼後再點。

**Q: 驗證碼過期了**
→ 驗證碼有效時間為 5 分鐘，過期後需重新查詢並產生。

**Q: 視窗看不到「驗證碼」和「確認 LINE 綁定」按鈕**
→ 視窗太小被裁切。Tab 1 右邊有捲軸，向下滾動即可。或把視窗拉大。

**Q: 解除綁定時出現 `name 'recno_hash' is not defined` 錯誤**
→ 舊版 bug（已修復）。現版本會自動從 recno 重新計算 recno_hash。

**Q: 本地資料庫有綁定記錄但 LINE 雲端找不到**
→ 可能是「更新程式時未取到 recno_hash 也未成功在 LINE 綁定」的情況（2026-07-09）。
   解法：用「綁定管理」的「解除綁定」強制解除。系統會提示「雲端無有效綁定記錄，本地記錄已移除」。

## 技術資訊

- **主程式**：patdb_query.py（Python 3 + tkinter，4 個分頁捲動式布局）
- **API 位置**：https://lineid-code.zeabur.app
- **日誌位置**：本資料夾下的 `logs/YYYY-MM-DD.log`
- **必要套件**：dbfread、requests
- **patdb 編碼**：cp950（舊診所系統格式）
- **本地資料庫**：SQLite `bindings.db`（自動產生，重灌時可刪除重建立）

### 三支主要 API（由 lineid_code Express 服務提供）

| API | 用途 | 認證 |
|-----|------|------|
| `POST /api/create-verify-code` | 產生 6 位驗證碼 | 無 |
| `POST /api/verify` | LINE Bot 驗證碼驗證 + 建立綁定 | LINE |
| `GET /api/admin/links-by-recno-hash` | 櫃台查綁定 link_id | `x-unbind-api-key` header |
| `POST /api/admin/unbind` | 櫃台強制解除綁定 | `x-unbind-api-key` header |

### 重要時序

```
櫃台                    patdb_query.py          Express API            LINE Bot
 │                            │                    │                       │
 │ 點確認並產生驗證碼          │                    │                       │
 │──────────────────────────>│                    │                       │
 │                            │ create-verify-code │                       │
 │                            │──────────────────>│                       │
 │                            │ {code, expiresAt}  │                       │
 │                            │<──────────────────│                       │
 │ 驗證碼已產生                │                    │                       │
 │<──────────────────────────│                    │                       │
 │                            │ 自動檢查 LINE 狀態   │                       │
 │                            │ links-by-recno-hash│                       │
 │                            │──────────────────>│                       │
 │  [等待 LINE 綁定] 按鈕發亮  │                    │                       │
 │<──────────────────────────│                    │                       │
 │                            │                    │                       │
 │ 告知病人去 LINE 輸入驗證碼                              病人輸入驗證碼     │
 │                                                           │              │
 │                                                           │ verify        │
 │                                                           │─────────────>│
```
